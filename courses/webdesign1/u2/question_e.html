Frågor inför HTML och CSS

  1. Fråga E1: Vad menas med kaskad i CSS? Hur fungerar det?
  2. Fråga E2: I HTML-taggarna kan man ha attribut med id och class och sedan referera till dem i CSS-koden. Vad är skillnaden, när ska man använda id och när ska man använda class?

Mitt svar

Svar på E1

I CSS så är det möjlig att det finns konflikter mellan olika definitioner av samma elementet. Om till exempel en regeln ska bestämmas att H3-taggen ska vara rött och sen en annan regeln bestämmer att H3-taggen ska vara grönt. Vilken regeln vinner då av den andra? Hur ska ett webbläsare veta vilken färg ska väljas om bara en färg kan användas. Därför är kaskad-tekniken introducerats. Kaskad är ett set av interna regler som bestämmer hur en set av motstridiga regler som skrevs i CSS ska lösas. I princip är det den regler som är specifika som vinner av de generella reglar då.

Svar på E2

Både ID och class (klass i Svenska) används så att det kan refereras mellan HTML-koden och CSS-koden. Skillnaden mellan ID och klass är att ID är unik och en klass är inte unik. Eftersom ID är unik kan varje element bara här ett ID och varje sida kan bara har ett element med detta ID. Klasser däremot kan användas flera gånger på flera element och olika klasser kan använda på samma elementet. En små fakta är att en element kan använda båda typer samtidigt. Klasser oftast användas för information kan återkommer medan information som är bara används en gång ska användas med ID. Till exempel, en sida med flera widgets passar klasser till.
